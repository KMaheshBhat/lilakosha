# 2026-06-30 a multi-inference provider based data synthesis

I had for the past couple of weeks, worked on the pipeline and did get the steps prepared for PIPPA ingestion, CDM normalization, and several refinements: character extraction, safety (sexuality, violence, toxicity) tagging, genre and theme tagging, and grammar (third-person past-tense) done for almost 3000 records.  All that, on the Gemma4 12B model running via llama.cpp on my local workstation.

The WSL instance crashed.  The pipelines though recoverable, the volume where I had mounted it was not - the `/tmp` as the name suggests was wiped off.

Thus I started again on the new SSD I had installed few weeks back, and I took the opportunity to rewrite the `Inference` contract, implement it for any OpenAI compatible provider, and then proceeded to test it against local (obviously), and other providers like OpenRouter, AI Horde, and Kilo Gateway.  The AI Horde did not work since it did not support structured output, but I am able to get the pipeline running now, as right at this moment, the dashboard reads:

```
INFO: ============================================================
INFO: Total Records Evaluated: 2000
INFO: Healthy Records:         498 (24.90%)
INFO: Defective Records:       1502
INFO: ============================================================
```

There are two pipelines running:
- `20260630202000-refine.yml` - on local against 501-1000 PIPPA sourced CDM records
- `20260630203000-refine.yml` - on OpenRouter against 1001-2000 PIPPA sourced CDM records

I currently estimate all of the PIPPA to get processed in about 30-45 days if I am able to keep scheduling these runs.

Once I get these runs stabilized, I will look at getting the Gutenberg selected works ingested to CDM records, and there - each book is going to be a mini project.

Stages/steps that still need work:

- restore and publish - the dataset is a deliverable.  I have packaged it as a `.jsonl` monolith.  I need to figure out how do I publish and restore from HuggingFace.
- prepare - the CDM records will need to be projected to what I have plans to train the models on.
- train and bake

Multi-provider inference has successfully decoupled pipeline throughput from local workstation availability. Future work can now optimize provider selection independently of pipeline orchestration.
